**Explanation:**
1. `WELCOME`, `LANGUAGE_SELECTION`, `REVIEW` are standard types defined in `SectionRegistry.ts` (keys/Ids) and `types.ts`, and also referenced when starting an initial state. They are part of the state machine definition schema.
2. `DONE` is used as a sentinel field ID indicating that a section's fields are complete. It isn't a hardcoded prompt text but a lifecycle state value.
3. `ADD_ANOTHER` is a sentinel field ID used in `isMultiItem` sections to prompt the user if they want to add another item. It isn't a string shown to the user, just a state tracker.
4. `CONFIRM_SKIP` is similarly a sentinel field ID representing the intermediate confirmation state when the user skips a required section. It tracks state logic.
